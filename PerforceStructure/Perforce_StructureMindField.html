Perforce MindField

Structure de dossier proposée pour la gestion des fichiers de production sur Perforce

-La gestion avec Perforce permet de pousser sur le serveur la dernière version travaillée.

-Nous n'attendons pas d’avoir une version finale pour soumettre le fichier sur le serveur.

-L’objectif est d’avoir un fichier non final qui évolue jusqu’à l’objectif du  livrable.

-Nul besoin d'incrémenter les versions, juste besoins de checkout lors du travail et de submit sur une base régulière.

-Ces processus permettent d’avoir un fichier sécurisé sur un serveur, de partager l’avancement de votre travail et d’avoir un fichier disponible et à jour pour toute collaboration.

Dossier Principaux sur le serveur

-Unreal_Build (Projet Unreal du film/jeu)

-Maya_Project_Animation (Projet maya pour les animateurs)

-Assets_Publish (Notre librairie)

En premier lieu:

Il y aura un Unreal_Build (dossier de projet unreal) principal  pour le projet. Ce dossier servira à l’intégration et l'assemblage du travail de tous les artistes. 

En deuxième lieu:

En parallèle nous allons demander au département d’animation et de shading d’avoir leur propre Unreal Projet personnel. Ce projet personnel ne doit pas être poussé sur Perforce. Si il est poussé, il sera supprimé. C’est votre propre management personnel.

**Peut être une scène test de groupe dans le P.**

Pour les animateurs:

C’est pour valider que le fbx fonctionne et pour la création du uassets d’anim. (Skeleton/Animation Sequence)

Pour les textures/shading: 

C’est pour valider les textures et pour la créations du uasset de matériel (Shader).

**uasset des textures + uasset de matériel**

Lighting département:

Le lighting departement y aura accès en temps et lieu, au préalable utiliser un projet test.

Workflow pour les modeleurs:


  1. Modéliser

  1. Si prescrit:
  1. Couper les UV (À même le logiciel de modélisation….)
  2. Créer le lightmap
  3. Assigner les Shader de base
  1. Exporter en fbx.
  2. Push le fbx dans la librairie perforce. (AssetsPublish/Meshes_Static)

**Vous pouvez garder vos scènes de modeling sur le P:. Elles ne doivent pas être push sur perforce.**
